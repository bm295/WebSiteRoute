# AI-Native Fintech Rewrite Documentation Set

This folder contains the initial documentation baseline required to rewrite this repository into an AI-native fintech platform.

## Included documents

1. **01-product-vision.md** – Product intent, user segments, and business goals.
2. **02-prd.md** – Product requirements document with scope and acceptance criteria.
3. **03-system-architecture.md** – Target architecture and service boundaries.
4. **04-domain-data-model.md** – Core fintech entities and storage strategy.
5. **05-api-contracts.md** – API surface for accounts, wallets, ledgers, payments, and risk.
6. **06-ai-operating-model.md** – How AI agents participate in product delivery and runtime operations.
7. **07-security-compliance.md** – Security, privacy, and regulatory control baseline.
8. **08-delivery-roadmap.md** – Milestones from foundation to production launch.
9. **09-agent-prompts-and-workflows.md** – Reusable prompts, checklists, and agent workflow templates.
10. **10-engineering-ways-of-working.md** – Team conventions for AI-native implementation.

## How to use

- Read in order for full context.
- Treat these files as the source of truth for the initial rewrite.
- Convert each section into actionable tickets before implementation.
